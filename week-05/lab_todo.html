Lab 05: To-Do Manager

In this lab, you'll combine your knowledge of OOP and the ability to read and write files using Python scripts. Up till now, your code runs once and then forgets everything it did until you run it again. But we can save some of that information in files as a makeshift database.

The goal: to create a script that manages your to-do list. It should be able to print a list of them to the terminal, to add new items to the list, and to mark items as completed.

Requirements

  • Make a class called Item. Objects of this class should have the following attributes:
    • A timestamp of when they were created.
    • A boolean marking the item as completed or not.
    • And the text of the actual to-do item.
  • Make a class called Manager. A single object of this class should be created when you run your script. It should do the following:
    • Print all of the to-do items in the list.
    • Add a new item to the list.
    • Mark an item as completed.
  • Each class should be in its own module which you will need to import accordingly. The main file should be called todos.py and is the only file to be run as a script.
  • Your to-do list should be stored in a file called todos.txt. You should never need to edit this file directly, the script should do that for you.

Things to Think About

  • You'll have to figure out how to store data in the file. Spend some time figuring this out. How will you retrieve the data from the file? How will it be organized so you can break the data into separate values? How can you convert that data into different Python types?
  • You've never worked with dates or times in Python before. Look up the datetime module in the standard library and see how it can help.
  • How will your script work? Will it expect arguments to be passed to it? Or will it run like an interactive program like our games?
